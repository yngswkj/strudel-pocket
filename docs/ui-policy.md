# Strudel Pocket UI Policy

Strudel Pocket はスマホ縦画面で素早く Strudel を書くためのアプリである。
機能追加時は、メイン画面を「コードを書く場所」として守ることを最優先にする。

## 基本方針

- メイン画面に新しい常設ボタンを増やさない。
- 新機能は原則として設定ドロワー、状態依存UI、既存タブ内に収める。
- 設定系、管理系、キャッシュ系、詳細表示は常駐させない。
- スマホ縦画面でエディタ領域と下部入力補助の見通しを優先する。
- エラー、ロード状況、貼付バー、数値フェーダーなどは必要な状態のときだけ表示する。

## 常時表示してよいもの

- 再生 / 停止
- 更新
- 設定
- コードエディタ
- 現在の入力モード
- 最低限の記号キー
- カーソル移動

## 状態に応じて表示するもの

- 数値フェーダー
- エラー表示
- サウンド貼付バー
- 音源ロード状況
- 検索候補

## ドロワーへ収めるもの

- 音源管理
- キャッシュ管理
- 表示設定
- 更新モード詳細
- お気に入り管理
- 視覚化設定

## Issue / PR 判断基準

新機能のIssueやPRでは、以下を確認する。

- メイン画面の常設UIを増やしていないか。
- 既存の設定ドロワーや既存タブに統合できないか。
- 表示が必要な状態でだけ出るUIにできないか。
- スマホ縦画面でエディタ領域を圧迫していないか。
- 機能説明文を常時表示して、操作面を狭くしていないか。
